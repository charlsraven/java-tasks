## выводы
дефолтная сериализация оказалось простой в использовании и эффективной из-за джава машины и эффективного кода
к минусам дефолтной можно только отнести неспособность сериализации static полей. 

кастомная реализация может приблизиться к резуальтатам дефолтной сериализации, 
она довольно гибкая по объёму информации и может выигрывать иногда за счёт этого, но неудобна в написании.

реализации с методами и exterlalizable оказались самыми неэффективными
к минусам exterlalizable можно, например, отнести неспособность десериализации final переменных. 
при использовании наследования, все классы-наследники, происходящие от externalizable-класса, должны иметь конструкторы 
по умолчанию.
