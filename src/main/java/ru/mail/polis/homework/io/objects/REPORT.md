## выводы
несмотря на простоту дефолтной сериализации, она выигрывает только по скорости сериализации (из-за работы умной 
джава машины) -- объёмы файлов вручную проще регулировать, а десериализация дефолной может занимать большое время 
из-за кучи служебной информации, которая нам может и не понадобиться, но реализуется интерфейсом, больше заточенным 
для работы с объектами, чем с примитивами.
к минусам дефолтной можно только отнести неспособность сериализации static полей. 

кастомная реализация более гибкая по объёму информации и может выигрывать иногда за счёт этого, но неудобна в написании.

exterlalizable сериализация и сериализация с методами показывают средние результаты.
к минусам exterlalizable можно, например, отнести неспособность десериализации final переменных. 
при использовании наследования, все классы-наследники, происходящие от externalizable-класса, должны иметь конструкторы 
по умолчанию.
